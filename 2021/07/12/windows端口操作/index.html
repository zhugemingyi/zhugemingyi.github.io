windows端口操作

发表于 2021-07-12 更新于 2021-07-13

每次电脑重启mysql都无法正常连接，查看端口发现是启动了多启动了一个mysqld.exe。应该是之前没卸载干净。先记录一下端口临时解决该问题的方法。

解决问题命令汇总

netstat -ano|findstr "3306"
// tasklist|findstr "5632"
taskkill /T /F /PID 5632

详细解析：

1.管理员身份运行cmd

2.查看所有运行的端口

netstat -ano

netstat 打印网络连接、路由表、连接的数据统计、伪装连接以及广播域成员；

netstat用法详解

3.查看端口对应的 PID

netstat -ano|findstr "3306"

查看端口3306对应的 PID

findstr 搜索文件中的文本模式

findstr用法详解

4.查看指定 PID 的进程

tasklist|findstr "5632"

tasklist 显示本地计算机或远程计算机上当前正在运行的进程列表

tasklist用法详解

5.结束进程

taskkill /T /F /PID 5632

强制（/F参数）杀死 pid 为 5632的所有进程包括子进程（/T参数）

taskkill 结束一个或多个任务或进程

taskkill用法详解
